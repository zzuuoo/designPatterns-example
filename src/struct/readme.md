## 结构类设计模式
+ 把类或对象结合在一起形成一个更大的结构。
### 已完成
+  代理模式
    + 为其他对象提供一个代理以便控制这个对象的访问。


### 待完成
+  适配器模式
    + 将一个类的方法接口转换成客户希望的另外一个接口。
+  装饰模式
    + 动态的给对象添加新的功能。
+  组合模式
    + 将对象组合成树形结构以表示“”部分-整体“”的层次结构。

+  亨元（蝇量）模式
    + 通过共享技术来有效的支持大量细粒度的对象。

+  外观模式
    + 对外提供一个统一的方法，来访问子系统中的一群接口。

+  桥接模式
    + 将抽象部分和它的实现部分分离，使它们都可以独立的变化。
